1. Introduction

This lecture focuses on Approximation in Value Space and Rollout Algorithms. These methods are crucial for solving large-scale problems where exact DP is intractable.

Why Finite Horizon?

  • Common in practice (e.g., MPC).
  • Advantageous for Rollout.
  • Deterministic nature eliminates need for Monte Carlo (simpler).

Key Applications

  • Integer Programming.
  • Scheduling & Routing.
  • Model Predictive Control (MPC).

2. Recall: Deterministic DP

Problem Formulation

System Dynamics: \( x_{k+1} = f_k(x_k, u_k) \)

Total Cost: \( J(x_0; u_0, \dots, u_{N-1}) = g_N(x_N) + \sum_{k=0}^{N-1} g_k(x_k, u_k) \)

Objective: Minimize total cost to find \( J^*(x_0) \).

The DP Algorithm

graph TD A[Start: Terminal Cost] -->|J*_N = g_N| B[Backward Induction] B -->|For k = N-1 to 0| C[Compute J*_k] C -->|Min over u_k| D[J*_k = min (g_k + J*_{k+1})] D -->|Until k=0| E[Optimal Cost J*_0]

3. Shortest Path Equivalence

A deterministic finite-horizon DP problem is mathematically equivalent to finding the Shortest Path in a layered graph.

graph LR S((Start x0)) --> A((x1)) S --> B((x1')) A --> C((x2)) A --> D((x2')) B --> D B --> E((x2'')) C --> T((Term)) D --> T E --> T style S fill:#dcfce7,stroke:#22c55e,stroke-width:2px style T fill:#fce7f3,stroke:#ec4899,stroke-width:2px

Key Insight

Nodes represent states \(x_k\). Edges represent controls \(u_k\) with weights equal to stage costs \(g_k\). Finding the optimal control sequence is exactly finding the path with minimum total weight.
