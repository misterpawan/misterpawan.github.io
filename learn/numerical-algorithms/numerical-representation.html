Number Systems

Base 10 (Decimal)

We write $1234.1234$ as:

Integer: $4\times 10^0 + 3\times 10^1 + 2\times 10^2 + 1\times 10^3$
Fraction: $1\times 10^{-1} + 2\times 10^{-2} + 3\times 10^{-3} + \dots$

Base $\beta$

Binary ($\beta=2$), Octal ($\beta=8$), Hex ($\beta=16$).

\[ (a_n\dots a_0.b_1\dots)_{\beta} = \sum_{k=0}^{n}a_k\beta^{k} + \sum_{k=0}^{\infty} b_k \beta^{-k} \]

Conversion Algorithms

Integer (Divide by 2)

Example: $(11)_{10}$ to Base 2

11 / 2 = 5 R 1 -> a_0 = 1
5 / 2 = 2 R 1 -> a_1 = 1
2 / 2 = 1 R 0 -> a_2 = 0
1 / 2 = 0 R 1 -> a_3 = 1
Result: (1011)_2

Fraction (Multiply by 2)

Example: $(0.625)_{10}$ to Base 2

2 * 0.625 = 1.25 -> b_1 = 1
2 * 0.25 = 0.50 -> b_2 = 0
2 * 0.50 = 1.00 -> b_3 = 1
Result: (0.101)_2

The Problem with Precision

Not all numbers terminate neatly. Consider $\frac{1}{5} = 0.2$ in decimal. In binary, it requires an infinite number of digits.

0.2 $\rightarrow$ 0.0011 0011 0011 ...

‚ö†Ô∏è Roundoff Error

This is roundoff error in its basic form. 0.2 is stored fine in base-10, but is an infinite repeating fraction in base-2, meaning computers cannot store it exactly.

Case Study: Intel Pentium FDIV Bug (1994)

A hardware bug in the floating point unit caused errors in division for specific numbers.

  • Discovery: Dr. Thomas Nicely (Lynchburg College) noticed computations were wrong.
  • Example: $1/824633702441.0$ was calculated incorrectly beyond the 8th digit.
  • Impact: Intel initially denied importance, stock fell, IBM halted shipments.
  • Cost: Intel set aside $420 Million to fix.
TO: Whom it may concern
RE: Bug in the Pentium FPU
DATE: 30 October 1994

It appears that there is a bug in the floating point unit... The Pentium FPU is returning erroneous values for certain division operations.

Rounding vs. Truncation

Roundoff Error

Occurs when digits are lost due to a limit on memory (e.g., storing 1/3 as 0.3333).

Truncation Error

Occurs when discrete values are used to approximate a mathematical expression (e.g., stopping a Taylor series at $n=10$).

Conditioning

  • Well-conditioned: Numerical results are insensitive to small variations in input.
  • Ill-conditioned: Small variations lead to drastically different results.
